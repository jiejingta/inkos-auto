# InkOS Auto — 使用指南

## 这个项目能帮我做什么
一句话说：它可以帮你持续写长篇小说，并自动做审查、修订、真相文件同步和章节管理。

典型场景：
- 我想从零开始搭一本书，让系统自动往下写
- 我想在网页里看章节、看日志、改提示词
- 我想让系统无人值守连续写作
- 我想把不满意的章节重写、重命名或批量改标题
- 我想看每一章的状态、真相文件和输出结果

## 使用前准备
- 电脑里需要有 `Node.js 20+`
- 这个项目使用 `pnpm` 工作区；如果你没有全局安装，也可以直接用 `npx pnpm`
- 需要准备至少一组可用的大模型接口配置
- 进入真正的小说项目目录后再执行 `inkos` 命令，这个目录里要有 `inkos.json`

## 快速开始
1. 在项目根目录执行 `npx pnpm build`
2. 进入你的小说项目目录
3. 配好 `.env` 或 `inkos.json` 里的模型信息
4. 执行 `inkos studio`
5. 打开网页后先确认书籍、章节和日志都能正常显示
6. 想自动写作时执行 `inkos up`

## 常见任务
### 我想启动网页工作台
1. 进入你的小说项目目录
2. 执行 `inkos studio`
3. 打开终端里显示的网址
预期结果：可以在网页里看到书籍、章节、日志、真相文件和提示词页面

### 我想让系统自动继续写
1. 进入小说项目目录
2. 执行 `inkos up`
3. 观察网页日志或 `inkos.log`
预期结果：系统会先检查历史章节是否已通过；如果发现章节显示已通过但真相文件还停在上一章，会先自动补同步，再继续写下一章

### 我想停止自动写作
1. 在同一项目目录执行 `inkos down`
2. 或者直接在运行中的终端里停止当前进程
3. 停止后先看一次 `inkos status --chapters`
预期结果：守护进程停止；如果刚好停在写章中途，最新一章可能需要再做一次同步或修复

### 我想批量修正章节标题
1. 进入小说项目目录
2. 执行 `inkos write retitle "<书名或bookId>" --all`
3. 如果只想试一段范围，可加 `--from` 和 `--to`
预期结果：章节标题会按 `book_rules.md` 里定义的风格重新审查，并同步到章节列表、章节正文首行和摘要投影

### 我想查看写作日志
1. 进入小说项目目录
2. 打开 `inkos.log`
3. 或在网页工作台的日志区查看最近日志
预期结果：可以看到当前阶段、报错信息、章节状态和自动修订过程

### 我想看每个 AI 环节实际发了什么、回了什么
1. 进入小说项目目录
2. 打开 `inkos-ai.log`
3. 或在 `inkos studio` 的“日志”页切到“AI 原文”
预期结果：可以看到每次请求的提示内容、所属环节、模型原始返回和报错信息

### 我想看某章为什么没通过
1. 执行 `inkos status --chapters`
2. 找到对应章节的状态和问题列表
3. 必要时再用网页工作台查看正文和真相文件
预期结果：能看到是审查没过、状态损坏，还是接口请求失败

### 我想在网页里手动通过章节，或驳回问题章节
1. 打开 `inkos studio`，进入对应书籍或章节页
2. 只对“待审核 / 审计失败”的章节点“通过”或“驳回”
3. 如果点“驳回”，先确认提示，因为这会回滚到上一章并丢弃后续章节
预期结果：网页里的“通过”现在会真正提交这一章的正式状态；不是只改一个显示标签

### 我想在网页里改真相文件
1. 打开书籍详情页里的“真相文件”
2. 选择文件并编辑内容
3. 保存后看页面上的同步提示
预期结果：除了文件本身被保存，系统还会按需要刷新记忆索引、状态镜像、事实历史或最新章节快照

## 输出说明
- `books/<书名或bookId>/chapters/`
  这里是每章正文文件。现在系统会自动清掉正文顶部残留的旧标题，避免一章开头出现两个标题。
- `books/<书名或bookId>/chapters/index.json`
  这里保存章节列表里的标题、字数、状态等信息。网页章节表主要读这里。
- `books/<书名或bookId>/story/`
  这里是真相文件，例如当前状态、伏笔、账本、卷纲等。
- `inkos.log`
  这是守护写作和流水线日志文件，适合排查问题。
- `inkos-ai.log`
  这是 AI 原文日志，里面会记录每个已接入环节实际发给模型的内容，以及模型原始返回或错误。

## 常见问题
**Q: 为什么标题会和我想象的不一样？**  
A: 章节标题会同时参考 `book_rules.md`、章节正文和历史标题。若标题风格要求没有写清楚，先补到 `book_rules.md` 再批量重命名。

**Q: 网页里“题材”页面的几个开关是什么意思？**  
A: `数值系统` 代表是否长期维护资源/账本；`战力体系` 代表是否重点审查力量成长和强弱逻辑；`时代考据` 代表是否需要更严格地核对时代背景和事实资料。

**Q: 为什么一章开头会出现两个标题？**  
A: 新版本已经会在落盘前自动清理旧标题残留。如果你看到旧章节里还有这个问题，可以跑一次批量重命名或重存该章。

**Q: 为什么日志里出现 429 / 529，但没有马上继续写下一章？**  
A: 现在这类瞬时请求错误会先在当前调用里自动重试；如果还失败，会等下一次调度再继续，而不是立刻把整章从头重跑。

**Q: 为什么我停掉 `inkos up` 之后，最新一章状态怪怪的？**  
A: 如果你在写章中途手动停掉，最新一章可能只写了一半流程。先看 `inkos status --chapters`，必要时再做同步或修复。

**Q: 为什么我之前看不到 `inkos.log`？**
A: 旧版本如果是从 Studio 页面启动，有可能只有网页实时事件，没有落项目根目录日志。新版本会在项目根目录同时维护 `inkos.log` 和 `inkos-ai.log`。

**Q: 为什么网页里的“通过”现在和以前不一样？**  
A: 现在网页里的“通过”会走完整提交流程。如果这章之前是“审计失败”，系统会一并提交暂存的真相文件、刷新快照和记忆索引，不再只是把状态改成“已通过”。

**Q: 为什么章节列表显示第 10 章已通过，但真相文件还停在第 9 章？**
A: 这是旧版本可能留下的状态不同步。新版本在继续写下一章前会自动检查并补同步：把最新已通过章节重新结算到真相文件、摘要、情绪弧线、快照和记忆索引里，再往后写。

**Q: `chapter_summaries.md` 或 `emotional_arcs.md` 只停在很早的章节怎么办？**
A: 新版本会在 `inkos write sync` 或继续自动写作前检查缺口。如果摘要缺第 2-9 章，或情绪弧线长期停在第 1 章，会先读取这些章节正文补回对应记录，再继续同步最新章节。

**Q: 为什么我在网页里驳回“审计失败”的最新章节，之前会提示失败？**  
A: 老项目里如果上一章快照丢失，旧版本会直接报错。现在网页驳回会优先尝试正常回退；如果发现缺快照，但当前坏掉的是“最新那一章”，系统会先裁掉这章，再按上一章正文自动补一次状态同步，不需要你先手工补快照。

## 已知限制
- 章节标题风格主要还是依赖 `book_rules.md` 写得是否清楚
- 如果模型本身连续输出低质量正文，系统仍可能进入多轮修订，写作速度会明显变慢
- 中途强行停止写作时，最新一章仍可能留下半完成状态
